Queries for Customers  ¶

The resource supports

  • Conditional GET

  • Paging: the response is split into pages of data

This route should not be accessed directly, but by posting a query and navigating to the result url.

Access a query for Customers
GET/customerapi/queries{?PageSize,PageOffset,SortBy,Filter}

Gets the list of Customer. Only the Customer that are accessible for the user specified by the authorization header are returned. NOTE: if the result set is empty, an empty collection is returned with a normal 200 HTTP code.

Example URI

GET http://polls.apiblueprint.org//customerapi/queries?PageSize=20&PageOffset=0&SortBy=0&Filter=
URI Parameters
HideShow
PageSize
number (optional) Default: 20 Example: 20

The number of Customers per page. The maximum value is 100.

PageOffset
number (optional) Default: 0 Example: 0

The number of the Customer which will be the first in the result list.

SortBy
string (optional) Default: 0 Example: 0

A list of propertys by which the result list is sorted. “+” indicates ascending “-” descending

Filter
string (optional) 

Filter collection by Customer properties
